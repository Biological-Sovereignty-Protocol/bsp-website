# Roadmap do BSP

O caminho para a soberania biológica universal.

## Concluído

### Fase 0 — Fundação
**2024 Q3**

- Especificação do BEO e IEO redigida
- Esquema de assinatura Ed25519 definido
- Modelo de armazenamento Arweave projetado
- Protótipos iniciais de contratos SmartWeave

### Fase 1 — Protocolo Core v1
**2025 Q1**

- Contratos de registro e gerenciamento de BEO implantados na Arweave
- Contratos de registro de IEO implantados
- Emissão e revogação de ConsentToken
- BSP Registry API (Node.js) — alpha pública

### Fase 2 — SDK e Ferramentas para Desenvolvedores
**2025 Q2–Q3**

- SDK TypeScript (`bsp-sdk`) publicado no npm
- SDK Python (`bsp-sdk`) publicado no PyPI
- Ferramentas CLI para gerenciamento de BEO
- Site de documentação BSP lançado
- Exchange API para submissão de BioRecord

### Fase 3 — Congelamento da Especificação
**2025 Q4**

- Especificação BSP v1.0 finalizada
- Processo BIP (BSP Improvement Proposals) estabelecido
- Auditoria de segurança concluída
- Implantação BSP v1.0 na mainnet

## Em Desenvolvimento

### Fase 4 — Expansão do Ecossistema
**2026**

- Integrações com dispositivos wearables (Apple Health, Garmin, Oura)
- Conectores para laboratórios de genômica (23andMe, Nebula, Sequencing.com)
- App móvel BSP Identity
- Padrão HealthRecord (BioRecord v2)

### Fase 5 — Governança e Descentralização
**2026–2027**

- Mecanismo de votação on-chain para BIPs
- Upgrades de protocolo multi-sig
- Nós de registro descentralizados
- Âncoras BSP cross-chain (Ethereum, Solana)

### Fase 6 — Mercados de Dados de IA
**2027+**

- Marketplace de dados biológicos consentidos
- Pools de datasets para treinamento de IA (opt-in)
- Produtos de dados de coorte longitudinal
- Compartilhamento de receita para contribuidores de dados

---

*O desenvolvimento do protocolo é orientado pela comunidade e sujeito a mudanças através do processo BIP.*
