# Roadmap de BSP

El camino hacia la soberanía biológica universal.

## Completado

### Fase 0 — Fundación
**2024 Q3**

- Especificación de BEO e IEO redactada
- Esquema de firma Ed25519 definido
- Modelo de almacenamiento Arweave diseñado
- Prototipos iniciales de contratos SmartWeave

### Fase 1 — Protocolo Core v1
**2025 Q1**

- Contratos de registro y gestión de BEO desplegados en Arweave
- Contratos de registro de IEO desplegados
- Emisión y revocación de ConsentToken
- BSP Registry API (Node.js) — alpha pública

### Fase 2 — SDK y Herramientas para Desarrolladores
**2025 Q2–Q3**

- SDK TypeScript (`@bsp/sdk`) publicado en npm
- SDK Python (`bsp-sdk`) publicado en PyPI
- Herramientas CLI para gestión de BEO
- Sitio de documentación BSP lanzado
- Exchange API para envío de BioRecord

### Fase 3 — Congelamiento de la Especificación
**2025 Q4**

- Especificación BSP v1.0 finalizada
- Proceso BIP (BSP Improvement Proposals) establecido
- Auditoría de seguridad completada
- Despliegue BSP v1.0 en mainnet

## En Desarrollo

### Fase 4 — Expansión del Ecosistema
**2026**

- Integraciones con dispositivos wearables (Apple Health, Garmin, Oura)
- Conectores para laboratorios de genómica (23andMe, Nebula, Sequencing.com)
- App móvil BSP Identity
- Estándar HealthRecord (BioRecord v2)

### Fase 5 — Gobernanza y Descentralización
**2026–2027**

- Mecanismo de votación on-chain para BIPs
- Actualizaciones de protocolo multi-sig
- Nodos de registro descentralizados
- Anclas BSP cross-chain (Ethereum, Solana)

### Fase 6 — Mercados de Datos de IA
**2027+**

- Marketplace de datos biológicos consentidos
- Pools de datasets para entrenamiento de IA (opt-in)
- Productos de datos de cohorte longitudinal
- Reparto de ingresos para contribuidores de datos

---

*El desarrollo del protocolo es impulsado por la comunidad y sujeto a cambios a través del proceso BIP.*
